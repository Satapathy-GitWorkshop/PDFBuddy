â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€

ERROR: "EADDRINUSE: address already in use :::5000"
  FIX:  Port 5000 is busy. Either:
        a) Kill the process using it: lsof -ti:5000 | xargs kill
        b) Change PORT in backend/.env to 5001

ERROR: "CORS error" in browser console
  FIX:  Make sure FRONTEND_URL in backend/.env matches your frontend URL exactly.
        Example: FRONTEND_URL=http://localhost:3000 (no trailing slash)

ERROR: "Cannot find module 'pdf-lib'"
  FIX:  Run: cd backend && npm install

ERROR: "LibreOffice conversion failed"
  FIX:  Install LibreOffice (see Section 6A above).
        Verify with: libreoffice --version

ERROR: "File too large" (413 error)
  FIX:  Increase MAX_FILE_SIZE_MB in backend/.env

ERROR: "npm install" fails on Windows with "node-gyp" errors
  FIX:  Install Windows build tools:
        npm install -g windows-build-tools
        Or use WSL2 (Windows Subsystem for Linux)

ERROR: Puppeteer fails to launch
  FIX:  On Linux: sudo apt-get install -y chromium-browser
        Set env: PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser

ERROR: Frontend shows blank page
  FIX:  Open browser console (F12). Check for errors.
        Usually means Vite proxy is not connecting to backend.
        Make sure backend is running first.

TIP 1: Use split terminal
  Run backend and frontend side-by-side in your terminal.
  VS Code has a built-in split terminal (Ctrl+Shift+5).

TIP 2: Hot reload is enabled
  Backend (nodemon): auto-restarts when you edit any .js file
  Frontend (Vite):   instantly updates in browser when you save

TIP 3: Check backend logs
  All requests are logged by Morgan middleware.
  Watch the backend terminal to debug API issues.

TIP 4: Uploaded files location
  Files uploaded by users go to: backend/uploads/
  Processed output files go to:  backend/outputs/
  These are auto-cleaned every 30 minutes.

TIP 5: Add a new PDF tool
  1. Create route:    backend/routes/yourtool.js
  2. Register route:  add to backend/server.js
  3. Add API call:    frontend/src/utils/api.js
  4. Create page:     frontend/src/pages/YourTool.jsx
  5. Add route:       frontend/src/App.jsx
  6. Add card:        frontend/src/pages/Home.jsx

TIP 6: Tailwind CSS classes
  The app uses Tailwind CSS for styling.
  Custom colors defined in frontend/tailwind.config.js:
    brand-red:  #E8302A  (primary red)
    brand-dark: #1a1a2e  (dark text)
